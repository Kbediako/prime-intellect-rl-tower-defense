### Overview
- **Environment ID**: `kbediako/tower_defence`
- **Python package**: `prime_td_env`
- **Short description**: Tower defense environments for legacy macro-round hosted RL and actual-game command replay/SFT.
- **Tags**: games, tower-defense, rl, verifiers, prime-rl, multi-turn
- **Published package shape**: `env.py`, `pyproject.toml`, `README.md`, `src/`
- **Current primary route**: `actual_game_command` replay/eval and adapter training; legacy `macro_round` remains available for historical hosted RL work.

### Datasets
- **Legacy dataset**: Procedural tower-defense seeds generated by the macro-round environment.
- **Actual-game dataset**: Packaged replay corpora under `src/prime_td_env/data/actual_game_command/{train,eval,test}_examples.json`.
- **Snapshot mode**: Configurable round snapshots (`dataset.snapshots`) for curriculum and stability studies.
- **Split control**: Training/eval sample volume is controlled by environment args and run config.

### Task
- **Legacy type**: Multi-turn game interaction (macro-round planning).
- **Legacy interaction contract**: One assistant plan maps to one in-game round progression.
- **Legacy action surface**: Candidate-index planning with `{"type":"plan","actions":[{"type":"choose","index":N}, ...]}`.
- **Actual-game type**: Offline single-decision replay over the multiplayer bridge command surface.
- **Actual-game wrapper(s)**: `wrapper="actual_game_command"` or `wrapper="benchmark_commands"`.
- **Actual-game action surface**: Exactly one JSON object with `{"kind":"command"|"noop","commandType":"place_tower|sell_tower|upgrade_tower|trigger_next_round","payload":{...},"reason":"..."}`.

### Quickstart
Run local smoke checks:

```bash
PYTHONPATH=src python3 scripts/smoke.py
```

Run baseline local evaluation:

```bash
PYTHONPATH=src python3 scripts/eval_baseline.py --episodes 10 --max-rounds 20 --output out/metrics.json
```

Run hosted training:

```bash
prime rl run configs/lab/prime-td.toml
```

Run actual-game gold replay evaluation on the packaged holdout:

```bash
PYTHONPATH=src python3 scripts/run_actual_game_command_eval.py --mode gold
```

Run actual-game live evaluation against a served model:

```bash
PYTHONPATH=src python3 scripts/run_actual_game_command_eval.py --mode inference --model Qwen/Qwen3-4B-Instruct-2507
```

Export packaged actual-game train/eval examples to chat-style SFT JSONL:

```bash
PYTHONPATH=src python3 scripts/export_actual_game_command_sft_dataset.py --examples src/prime_td_env/data/actual_game_command/train_examples.json --output artifacts/actual_game_command_sft/train_messages.jsonl --report artifacts/actual_game_command_sft/train_export_report.json
PYTHONPATH=src python3 scripts/export_actual_game_command_sft_dataset.py --examples src/prime_td_env/data/actual_game_command/eval_examples.json --output artifacts/actual_game_command_sft/eval_messages.jsonl --report artifacts/actual_game_command_sft/eval_export_report.json
```

Train an actual-game LoRA adapter locally once the train/eval JSONL exports exist:

```bash
PYTHONPATH=src python3 scripts/train_actual_game_command_sft.py --model Qwen/Qwen3-4B-Instruct-2507 --train-jsonl artifacts/actual_game_command_sft/train_messages.jsonl --eval-jsonl artifacts/actual_game_command_sft/eval_messages.jsonl --train-examples-path src/prime_td_env/data/actual_game_command/train_examples.json --train-export-report artifacts/actual_game_command_sft/train_export_report.json --examples src/prime_td_env/data/actual_game_command/eval_examples.json --output-dir artifacts/actual_game_command_sft/adapter --report artifacts/actual_game_command_sft/train_report.json --eval-report artifacts/actual_game_command_sft/eval_report.json --eval-predictions-out artifacts/actual_game_command_sft/eval_predictions.jsonl --eval-rows-out artifacts/actual_game_command_sft/eval_rows.jsonl
```

### Environment Arguments
| Group | Key args | Purpose |
| --- | --- | --- |
| `wrapper` | `wrapper="macro_round"` or `wrapper="actual_game_command"` / `wrapper="benchmark_commands"` | Selects the legacy macro-round planner or the actual-game replay environment. |
| `difficulty` | `max_rounds` | Controls episode horizon/curriculum cap. |
| `observation` | `max_action_candidates`, `max_build_slots`, `max_towers`, `max_threats` | Bounds payload size and candidate space. |
| `candidate_balance` | `min_build_frac`, `max_upgrade_candidates`, `by_phase.*` | Tunes build/upgrade exposure by phase. |
| `dataset` | `policy`, `rollout_steps`, `snapshots`, `safe_explore_*` | Controls training observation generation. |
| `actual_game dataset` | `dataset.examples_path`, `dataset.examples_paths`, `num_examples`, `seed_start` | Selects packaged actual-game replay examples and sampled slices. |
| `actual_game rewards` | `reward_weights.format`, `reward_weights.env` | Blends strict JSON-format reward with exact-target replay reward. |
| `rules` | `auto_advance_round`, `prep_actions_*`, `mask_sell` | Governs turn semantics and allowed behavior. |

### Metrics
| Metric | Meaning |
| --- | --- |
| `reward/mean` | Aggregate training reward over samples in a step. |
| `metrics/num_turns` | Episode length in environment turns. |
| `format_reward` | Validity of structured plan/action output. |
| `macro_round_delta` (derived) | Round advance per turn; expected `delta_round == 1` after turn 1. |
| `env_pass_rate` (actual-game eval) | Exact target-match rate on the actual-game replay holdout. |
| `format_pass_rate` (actual-game eval) | JSON/schema-valid output rate on the actual-game replay holdout. |
| Action mix (derived) | Build vs upgrade distribution, tracked by phase. |

### Run Hygiene
- Record run IDs and config filenames immediately after launch.
- Pull rollouts for every sampled step and parse all user/assistant turns.
- Validate macro-round invariant `delta_round == 1` for user observations after turn 1.
- If sample upload 500s appear, reduce tokens/observation caps or horizon pressure.

Detailed longitudinal results and run-by-run analysis: `docs/RESULTS.md`.
